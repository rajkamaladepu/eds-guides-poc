Implementing SQL Always On availability groups

Table: Implementing SQL Always On availability groups introduces the procedures you must complete to use availability groups in your Enterprise Vault environment. You must complete these procedures in the order shown.
Table: Implementing SQL Always On availability groups
Step
Task
See this section for more details
Step 1
Stop all Enterprise Vault services, to ensure that Enterprise Vault does not write changes to the databases while you set up your availability group.
See Stopping Enterprise Vault services .
Step 2
Deploy an availability group to support your Enterprise Vault environment.
See Deploying a SQL Always On availability group .
Step 3
Find the configuration of your existing Enterprise Vault databases.
See Finding the existing database configuration using Get-EVDatabaseDetail .
Step 4
Reconfigure Enterprise Vault to connect to its databases, using the availability group's listeners.
See Reconfiguring Enterprise Vault using Set-EVDatabaseDetail .
Step 5
Restart all Enterprise Vault services.
See Starting Enterprise Vault services .
