Viewing expiry report

The expiry report lists all archived items from Enterprise Vault, whose retention period is about to end. It lists the items that are about to expire in 60 days. If required, you can restore selected items to your vault or mailbox.
Administrators can enable the Expiry Report option while creating a new search policy. By default, this option is disabled. Administrators can enable or disable the Expiry Report feature using the Allow Expiry Report to be viewed option on the Features tab of the Default Search Policy Properties dialog in the Enterprise Vault Administration Console.
Expiry report can be generated in two ways:
  • From Enterprise Vault Administration Console
  • From Enterprise Vault Search
From Enterprise Vault Administration Console, administrators can select the archive for which they want to generate the expiry report, and then right-click on the archive to select the View Expiry Report option. The expiry report gets loaded in Enterprise Vault Search.
When the Allow Expiry Report to be viewed option is enabled on the Features tab of the Default Search Policy Properties dialog, users can see the View Expiry Report option in Enterprise Vault Search. The expiry report can be generated for a specific archive or multiple archives from Enterprise Vault Search.
