Importing the retention folder information in the XML file into retention plans

After you have defined the retention folder information in an XML file, you can import it into one or more retention plans. Enterprise Vault comes with multiple PowerShell cmdlets with which you can configure retention plans and their associated retention folders. Table: PowerShell cmdlets for configuring retention plans and folders describes these cmdlets.
Table: PowerShell cmdlets for configuring retention plans and folders
Cmdlet
Description
Get-EVRetentionPlan
Returns the properties of an existing retention plan and its associated retention folders.
New-EVRetentionPlan
Creates a retention plan and imports into it the retention folder information that you have defined in an XML file.
Set-EVRetentionPlan
Updates the retention folder information that you have associated with an existing retention plan.
For example, the following command creates a retention plan in which the specified XML file, folders.xml, defines the attributes of one or more retention folders. The retention settings that you define for these folders in the XML file override the "5 years" retention category that you associate with the plan.
New-EVRetentionPlan -Name "Projects Retention Plan" -Description "Plan for adding retention folders" -RetentionCategory "5 years" -FolderXmlPath "c:\My XML Files\folders.xml"
For more information, see the PowerShell Cmdlets guide.
