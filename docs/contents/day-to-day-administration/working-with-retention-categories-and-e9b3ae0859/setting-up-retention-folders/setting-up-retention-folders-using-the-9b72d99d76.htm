Setting up retention folders using the Administration Console

You can define the required retention folders as part of the process of creating a new retention plan or changing the properties of an existing retention plan. For example, when you open the properties of an existing plan and then click the Folders tab, you can add, remove, and modify its associated retention folders.
When you add a new retention folder, you must specify the following:
  • The name of the retention folder, as it is to appear in the users' archives.
Optionally, you can specify multiple variants of the folder name to suit users who employ different languages.
  • Whether any subfolders of the retention folder inherit their retention settings from it or have their own settings.
  • A retention category to associate with the retention folder.
  • Whether the retention category that you have associated with the retention folder overrides the individual retention categories on the items in the affected folders.
The online Help for the Administration Console provides extensive information on how to set up retention folders with it.
