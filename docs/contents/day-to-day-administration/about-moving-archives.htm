About moving archives

Enterprise Vault's Move Archive wizard lets you move content from existing Domino and Exchange mailbox archives, and from Domino and Exchange journal archives, to new archives or existing archives in other vault stores.
You can use the wizard to move mailbox archives between vault stores in one site, and between two sites. This includes moves between two sites that are controlled by different Enterprise Vault directories.
Move Archive supports the move of journal archives only within an Enterprise Vault site.
Move Archive is designed to move one user, or a few users, between vault stores, or across servers or sites. If you need to move a large number of users, refer to the Enterprise Vault Performance Guide at https://www.veritas.com/docs/100000918 for information about the impact on performance.
Note:
Move Archive supports the move of archives to destination servers that run Enterprise Vault 11.0 or later.
Move Archive handles each move operation differently, depending whether the source and destination archives are associated with a mailbox at the time of the move. Move Archive considers an archive to be:
  • Active if it is associated with a mailbox, regardless of whether the mailbox is enabled for archiving
  • Inactive if it is not associated with a mailbox
Move Archive supports move operations for the following archive types:
  • Domino mailbox and journal archives
  • Exchange mailbox and journal archives
Move Archive does not support the following archive types:
  • File System Archiving (FSA) archives
  • SharePoint archives
  • Shared archives
  • Exchange public folder archives
  • Internet Mail archives
  • SMTP archives
Move Archive also prevents moves in the following circumstances:
  • The source archive exceeds its archive usage limit and the destination is a new archive.
  • The source archive is closed.
  • The source archive contains items that Discovery Accelerator has placed on legal hold.
  • The destination archive exceeds its archive usage limit.
When an archive exceeds its usage limit, you should increase the limit on the Archive Properties: Archive Usage Limit tab in the Administration Console.
