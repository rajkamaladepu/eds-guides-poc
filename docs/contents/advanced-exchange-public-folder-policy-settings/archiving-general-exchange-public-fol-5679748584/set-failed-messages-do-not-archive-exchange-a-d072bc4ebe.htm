Set failed messages 'Do Not Archive' (Exchange Archiving General setting)

Description
If an item cannot be archived, the default archiving task behavior is to reprocess the item on the next archiving run because such items can often be successfully archived on a second try.
This setting enables you to change the behavior so items that fail archiving are marked as Do Not Archive and thus are not reprocessed on the next archiving run.
Supported values
  • Off (default). Failed items are not marked as Do Not Archive.
  • On. Failed items are marked as Do Not Archive.
