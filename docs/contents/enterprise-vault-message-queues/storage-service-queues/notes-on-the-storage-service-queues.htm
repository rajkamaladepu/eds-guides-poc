Notes on the Storage service queues

  • Monitoring the storage archive queue will indicate the Storage service is processing items. If the number of items in this queue does not change for at least 30 minutes, there is likely to be a problem. Check the Enterprise Vault event log on the storage computer for any errors and, if necessary, restart the Storage service.
  • When all the vault stores on an Enterprise Vault server are in backup mode, the Storage Archive queue is not processed.
  • Monitoring the restore queue will indicate the number of restores required by the users. Again, if the number of items on the queue does not change, there is likely to be a problem.
