Remove server from intranet zone (Exchange Outlook setting)

Description
Removes the listed servers from the Internet Explorer local intranet zone.
The effect of this is that, unless you set up appropriate access, users are prompted for their user names and passwords when they search their archives or view or restore archived items.
The change applies to the current user only, so other users of the same computer are unaffected.
If the user does not have permission to modify Internet Explorer the security settings are not changed and no error is generated.
Supported values
  • A text string. Defines the computers to be removed from the Internet Explorer local intranet zone. The string can contain wildcard characters, domain names, DNS aliases, or IP addresses.
To specify multiple computers, separate the names using semicolons (;).
The syntax is as follows:
computer1;_computer2_...
Some examples of text strings are as follows:
    webserver.mycorp.com



    *.mycorp.com



    mywebserver;*.mycorp.com
