Use this page to monitor and manage the subtasks. Use the Search controls to filter the subtasks in the list.</p><p class="- topic/p ">For example, to list only unsuccessful subtasks whose archive names contain "John Doe"</p><ol class="- topic/ol "><li class="- topic/li "><p class="- topic/p ">Enter <b class="+ topic/ph hi-d/b ">John Doe</b> in the <b class="+ topic/ph hi-d/b ">search text contains</b> box.</p></li><li class="- topic/li "><p class="- topic/p ">Click <b class="+ topic/ph hi-d/b ">Deselect All</b> to clear the current selections.</p></li><li class="- topic/li "><p class="- topic/p ">Select <b class="+ topic/ph hi-d/b ">Warnings</b> , <b class="+ topic/ph hi-d/b ">Will retry</b> , and <b class="+ topic/ph hi-d/b ">Failed</b> under <b class="+ topic/ph hi-d/b ">Subtask status</b>.</p></li><li class="- topic/li "><p class="- topic/p ">Click <b class="+ topic/ph hi-d/b ">Search</b>.</p></li></ol><p class="- topic/p ">You can also click Clear All to clear existing search criteria before you start a new search.</p><p class="- topic/p ">For each subtask, this page lists information in the columns described in the following table. The page includes only the columns that are relevant to the types of subtasks that are displayed.</p><p class="- topic/p ">Archive Name</p><p class="- topic/p ">The name of the archive that the subtasks are associated with.</p><p class="- topic/p ">Index Volume Range</p><p class="- topic/p ">The range of sequence numbers of the items indexed in the index volume.</p><p class="- topic/p ">Items Added</p><p class="- topic/p ">For upgrade and rebuild subtasks, this shows the total number of items successfully processed.</p><p class="- topic/p ">For synchronization subtasks, this shows the number of items that have been indexed by the subtask.</p><p class="- topic/p ">Items Deleted</p><p class="- topic/p ">For synchronization subtasks, this shows the number of pending deletions and other previously orphaned index entries items that have been removed from the index volume by the subtasks.</p><p class="- topic/p ">Items With Errors</p><p class="- topic/p ">For upgrade, rebuild and synchronization subtasks, this shows the number of items that could not be retrieved, usually due to storage issues.</p><p class="- topic/p ">In the case of upgrade and rebuild subtasks, items with errors cause the subtask to wait for user input if you did not select the Ignore items that cannot be indexed option when you submitted the task.</p><p class="- topic/p ">Missing Items</p><p class="- topic/p ">For verification subtasks, this shows the number of items in the archive that Enterprise Vault has failed to index.</p><p class="- topic/p ">Extra Items</p><p class="- topic/p ">For verification subtasks, this shows the number of orphaned index entries it found in the index volume. These are index entries for items that do not exist in the archive.</p><p class="- topic/p ">Original Location</p><p class="- topic/p ">For change location subtasks, this shows the original location of the index volume.</p><p class="- topic/p ">Destination Location</p><p class="- topic/p ">For change location subtasks, this shows the destination location of the index volume.</p><p class="- topic/p ">Status</p><p class="- topic/p ">The subtask's status.</p><p class="- topic/p ">Progress</p><p class="- topic/p ">Progress. The bar shows subtask's progress. The color of the bar matches the category that subtask status belongs to. Active and Successful tasks have a green bar, Not Active subtasks have a yellow bar, and Unsuccessful subtasks have a red bar.</p><p class="- topic/p ">Select subtasks from the list and use the buttons on the button bar or the items on the Actions menu to initiate the following actions:</p><ul class="- topic/ul "><li class="- topic/li "><p class="- topic/p ">Switch back to parent task summary view (button bar only). Click to show the task view.</p></li><li class="- topic/li "><p class="- topic/p ">Start. Click to start stopped subtasks.</p></li><li class="- topic/li "><p class="- topic/p ">Stop. Click to stop running subtasks.</p></li><li class="- topic/li "><p class="- topic/p ">Restart. Click to restart subtasks.</p></li><li class="- topic/li "><p class="- topic/p ">Retry. Click to retry failed subtasks.</p></li><li class="- topic/li "><p class="- topic/p ">Delete. Click to delete subtasks.</p></li><li class="- topic/li "><p class="- topic/p ">I have copied the files or Process waiting subtasks.</p></li></ul><p class="- topic/p ">For change location tasks, click I have copied the files when you have copied the files associated with the subtask to their new location.</p><p class="- topic/p ">For all other subtasks, click Process waiting subtasks to process the waiting subtasks.</p><ul class="- topic/ul "><li class="- topic/li "><p class="- topic/p ">Open report. Click to open the report file associated with the subtask.</p></li><li class="- topic/li "><p class="- topic/p ">Save report. Click to save the report file associated with the subtask.</p></li><li class="- topic/li "><p class="- topic/p ">Details. Click to display details about the selected subtask.</p></li><li class="- topic/li "><p class="- topic/p ">Copy selection to clipboard (Actions menu only).
