Deletion of upgraded index volumes

During the upgrade of index volumes, Enterprise Vault deletes the old 32-bit or 64-bit index volumes as soon as it makes replacement Elasticsearch index volumes live and searchable.
In the case of the upgrade of multiple index volumes that are associated with one archive, the upgrade might create fewer Elasticsearch index volumes than there were 32-bit or 64-bit index volumes. When a new Elasticsearch volume is made live and searchable, Enterprise Vault deletes the 32-bit or 64-bit ndex volumes that the Elasticsearch index volume replaces. If the upgrade does not encounter any items that cannot be indexed, it continues until all the 32-bit or 64-bit index volumes have been replaced by Elasticsearch index volumes.
Each time you run the Upgrade wizard, you can choose to ignore items that cannot be indexed during the upgrade. If you choose to ignore these items, even if the upgrade encounters items that cannot be indexed it proceeds to completion in the same way.
If you do not choose to ignore items that cannot be indexed, the upgrade pauses at the end of processing for each 32-bit or 64-bit index volume in which it encounters these items and waits for your input in the Monitor Indexing Tasks page.
