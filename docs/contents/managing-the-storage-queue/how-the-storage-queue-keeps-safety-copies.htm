How the Storage queue keeps safety copies

When the safety copies are kept on the Storage queue, the mailbox archiving process is as follows:
  1. An item is marked as archive pending.
  2. The item is added to .EVSQ file in the Storage queue location.
  3. Storage takes the item from the .EVSQ file and adds it to the appropriate archive.
  4. Enterprise Vault changes the archive pending item in the mailbox to a shortcut.
  5. Enterprise Vault scans to make sure that each SIS part and DVS file have been secured.
  6. When all the archived versions of all the items in the .EVSQ file are secured, Enterprise Vault deletes the .EVSQ file.
Enterprise Vault creates empty .EVSQ files and then populates them as required. The Storage queue location may contain files with zero size. Such files are part of the normal operation. The files are deleted when they are no longer needed.
