Maximum classification content size (Site Properties Storage setting)

Description
Specifies the maximum size in megabytes of the files that Enterprise Vault submits to the Veritas Information Classifier for classification. When a file exceeds this limit, Enterprise Vault automatically splits it into files that are of the specified size, and classification then proceeds across the set of files.
You may want to increase the limit if you have a complex classification policy that fails to match items because different parts of it match different files in the set. For example, this issue can arise if you have a policy that searches for both of the words fraud and corruption , when the first word is in one file and the second word is in another.
Note:
This setting is for use with the Veritas Information Classifier in Enterprise Vault 12.2 and later. It has no effect on classification using the Microsoft File Classification Infrastructure (FCI). For guidelines on how to configure the maximum content size for FCI classifications, see the Classification using the Microsoft File Classification Infrastructure guide.
Supported values
An integer specifying the maximum number of megabytes. The default is 32, and the minimum is 1.
