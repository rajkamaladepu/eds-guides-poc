Send Email Timeout (site properties IMAP setting)

Description
Defines the number of seconds that the Client Access Provisioning task waits for confirmation that newly-provisioned users have received the IMAP notification email.
The email is sent as soon as the task runs. If the specified time period elapses, and no confirmation of receipt arrives, the client access provisioning task report shows the error message Error sending notification: The operation has timed out. The process is repeated the next time the task is run.
Supported values
  • An integer in the range 1 to 300 that sets the duration. The default is 30.
